# TeraMotors Deployment Guide

## Problem Summary

The invoice PDF generation functionality was not working on Vercel because Puppeteer requires a full Chrome/Chromium browser instance, which cannot run in Vercel's serverless environment due to:
- Memory and execution time constraints
- Missing Chrome/Chromium binaries
- Serverless function limitations

## Solution

We've moved PDF generation to a separate Express server that can run on a traditional hosting platform (like Render.com, Railway, or any VPS) where Puppeteer can function properly.

## Architecture

```
┌─────────────────┐
│  Next.js App    │
│  (Vercel)       │──────────┐
└─────────────────┘          │
                             │ HTTP Requests
┌─────────────────┐          │ for PDF Generation
│  Express Server │◄─────────┘
│  (Render.com)   │
│  - Socket.io    │
│  - Puppeteer    │
│  - PDF Gen      │
└─────────────────┘
```

## Deployment Steps

### 1. Deploy the Frontend (Next.js) to Vercel

The frontend is already configured to use Vercel. Just make sure to set the environment variable:

**Vercel Environment Variable:**
```
NEXT_PUBLIC_SOCKET_URL=https://your-server-url.onrender.com
```

### 2. Deploy the Express Server to Render.com

#### Option A: Using Render Dashboard

1. Go to [Render.com](https://render.com) and create a new **Web Service**
2. Connect your GitHub repository
3. Configure the service:
   - **Name**: `teramotors-server`
   - **Root Directory**: `server`
   - **Build Command**: `npm install && npx puppeteer browsers install chrome`
   - **Start Command**: `npm start`
   - **Environment Variables**:
     ```
     NODE_ENV=production
     MONGODB_URI=mongodb+srv://valmonzer_db_user:f2Ju76Vku4eA73az@cluster0.lydxxdo.mongodb.net/?retryWrites=true&w=majority&appName=Cluster0
     CORS_ORIGIN=https://your-vercel-app.vercel.app
     PORT=5000
     ```

#### Option B: Using render.yaml (Recommended)

Create a `render.yaml` file in the project root:

```yaml
services:
  - type: web
    name: teramotors-server
    env: node
    rootDir: server
    buildCommand: npm install && npx puppeteer browsers install chrome
    startCommand: npm start
    envVars:
      - key: NODE_ENV
        value: production
      - key: PORT
        value: 5000
      - key: MONGODB_URI
        sync: false  # Set this in Render dashboard
      - key: CORS_ORIGIN
        sync: false  # Set this in Render dashboard
```

Then connect your repository to Render and it will automatically detect the `render.yaml` configuration.

### 3. Alternative Hosting Options

If you prefer other hosting platforms:

#### Railway.app
```bash
# Install Railway CLI
npm install -g @railway/cli

# Deploy
cd server
railway login
railway init
railway up
```

#### DigitalOcean App Platform
1. Create a new App
2. Select your GitHub repository
3. Configure:
   - **Source Directory**: `server`
   - **Build Command**: `npm install && npx puppeteer browsers install chrome`
   - **Run Command**: `npm start`

#### Traditional VPS (Ubuntu)
```bash
# Install Node.js
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# Install Chrome dependencies
sudo apt-get install -y chromium-browser

# Clone and setup
git clone your-repo
cd teramotors/server
npm install
npx puppeteer browsers install chrome

# Use PM2 to keep it running
sudo npm install -g pm2
pm2 start server.js --name teramotors-server
pm2 save
pm2 startup
```

## Troubleshooting

### Server Issues

**MongoDB Connection Error:**
- Verify `MONGODB_URI` is correct
- Check MongoDB Atlas IP whitelist (allow all: `0.0.0.0/0`)

**Puppeteer Errors:**
- Ensure Chrome is installed: `npx puppeteer browsers install chrome`
- On Render, ensure build command includes Chrome installation

**CORS Errors:**
- Verify `CORS_ORIGIN` matches your Vercel URL exactly
- Include protocol (`https://`) in the URL

### Frontend Issues

**PDF buttons not working:**
- Check browser console for errors
- Verify `NEXT_PUBLIC_SOCKET_URL` environment variable is set
- Ensure server is accessible from the browser

**Environment variables not loading:**
- Prefix must be `NEXT_PUBLIC_` for client-side access
- Restart Next.js dev server after changing `.env.local`
- Redeploy to Vercel after updating environment variables
